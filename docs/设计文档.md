## 1. 项目概述

### 1.1 目标

提供一个轻量级桌面工具，用户选择一个包含图片的文件夹后，工具自动解析图片的拍摄时间，并将文件夹重命名为统一的归档格式：

```
YYYY_MM_DD_城市_自定义标题
```

### 1.2 用户场景

用户手中有大量按日期/地点/事件组织的图片文件夹，但命名混乱。希望在归档时能统一重命名为可读、可搜索的标准格式，降低后续检索成本。

### 1.3 核心流程

1. 用户选择一个文件夹（或拖拽到工具界面）
2. 工具扫描文件夹中所有图片文件
3. 工具从图片 EXIF 中提取拍摄时间
4. 工具智能推断该组图片的代表日期
5. 用户填写 **地点** 和 **自定义标题**
6. 工具将文件夹重命名为 `YYYY_MM_DD_城市_自定义标题`
7. 工具输出操作日志

## 2. 技术方案

### 2.1 推荐方案：Tauri v2

| 维度 | 选择 | 理由 |
|------|------|------|
| 框架 | **Tauri v2** | 二进制体积 ~5MB（Electron ~150MB），原生文件对话框、文件操作、窗口管理 |
| 前端 | **React + TypeScript + Vite** | 生态成熟，Tauri 官方脚手架直接支持 |
| EXIF 解析 | **`kamadak-exif`** (Rust crate) | 原生 Rust 库，无 Node.js 依赖，性能好 |
| 设计系统 | **Claude Design System** | 奶油画布 + 暖珊瑚主色 + 深色产品表面 |
| CSS | vanilla CSS / CSS Modules | 轻量，不引入重型框架 |
| 图标 | **lucide-react** | svg 图标库，与 Claude 清爽风格一致 |
| 包管理 | **pnpm** | 节省磁盘，比 npm/yarn 快 |

### 2.2 为什么不是其他方案

| 方案 | 排除理由 |
|------|----------|
| Electron | 打包体积太大（~150MB），违背轻量化目标 |
| Python + PyQt | UI 风格老旧，无法实现 Claude 设计语言 |
| Python + 本地 Web | 需要额外启动服务，分发多一层依赖 |
| 纯前端 PWA | 无法直接操作本地文件系统（文件夹选择受限，无法重命名） |

### 2.3 为什么 Tauri 足够轻量

- 安装包体积：~5MB（vs Electron ~150MB）
- 内存占用：~50-80MB（vs Electron ~150-300MB）
- 启动速度：毫秒级
- 开发依赖：Rust + Node.js 双工具链（一次安装长期使用）
- 打包分发：一个 exe 文件，无需运行时

## 4. 核心逻辑设计

### 4.1 扫描与过滤（Rust 后端）

```
src-tauri/src/
├── scanner.rs      # 递归扫描，过滤图片扩展名
├── exif.rs         # 读取 EXIF 日期字段
├── inference.rs    # 日期推断（众数 → 最早）
└── renamer.rs      # 重命名，冲突检测
```

- 支持的扩展名：`.jpg` `.jpeg` `.png` `.tiff` `.tif` `.webp` `.heic` `.heif`
- 使用 `walkdir` crate 递归遍历

### 4.2 日期提取

优先级链（Rust 端实现）：

1. `kamadak-exif` 读取 **DateTimeOriginal** — 最可靠
2. 回退到 **DateTimeDigitized**
3. 回退到 **DateTime**
4. 回退到文件系统 `mtime`
5. 回退到文件名正则 `IMG_20240315_*.jpg` 提取

### 4.3 日期推断

```
众数(Mode) 优先 → 并列选最早 → 单图直接取 → 跨度>30天时传回前端让用户选
```

- 日期粒度仅到日（YYYY-MM-DD）
- 推断在 Rust 后端完成，结果传到前端

### 4.4 文件夹命名

```
YYYY_MM_DD_城市_自定义标题
```

- 过滤非法字符：`\ / : * ? " < > |` → 替换为 `-`
- 超过 MAX_PATH（Windows 255）时截断标题
- 重名时追加序号 `_001`

### 4.5 重命名

- Tauri 调用 `std::fs::rename`
- 只改文件夹名，不改内部文件

## 6. 开发路线图

### Phase 1 — 项目脚手架 + Rust 后端核心

| 步骤 | 内容 | 产出 |
|------|------|------|
| 1.1 | `npm create tauri-app` 初始化，选 React + TypeScript + Vite | 可运行的 Tauri 项目骨架 |
| 1.2 | 配置 `tauri.conf.json`：窗口标题、尺寸(540×640)、无边框/圆角 | 窗口配置就绪 |
| 1.3 | 编写 Rust 端 `scanner.rs`：递归遍历目录，`.jpg/.png` 等扩展名过滤 | 扫描函数 `scan_folder(path) -> Vec<ImageEntry>` |
| 1.4 | 编写 Rust 端 `exif.rs`：用 `kamadak-exif` 读取 DateTimeOriginal，降级 DateTime → mtime | 日期提取函数 `extract_date(path) -> Option<NaiveDate>` |
| 1.5 | 编写 Rust 端 `inference.rs`：众数推断 + 并列选最早 + 跨度检测 | 推断函数 `infer_date(dates) -> InferResult` |

| 1.7 | Tauri `#[tauri::command]` 暴露以上 4 个函数给前端 | 前后端 IPC 通道就绪 |

## 9. 约束与风险

### 9.1 约束

| 约束 | 说明 |
|------|------|
| 目标平台 | 仅 Windows（首发），Tauri 本身跨平台 |
| HEIC 支持 | `kamadak-exif` 不支持 HEIC，需要额外处理 |
| 路径长度 | Windows MAX_PATH=255 字符，需截断 |
| Rust 环境 | 首次开发需要安装 Rust 工具链 |

### 9.2 风险与缓解

| 风险 | 缓解 |
|------|------|
| 重命名不可逆 | 预览确认 + 日志记录原名称，支持手动回滚 |
| 大文件夹卡 UI | Rust 端 `async` 命令 + 前端 loading 状态 |
| 日期不匹配预期 | UI 手动覆盖 + 离群标记 |
